# 单位属性配置表

## 配置系统设计

### 重要原则

⚠️ **所有单位必须通过统一的配置结构进行属性配置**：

1. **统一配置接口**：所有单位使用相同的配置API和数据结构
2. **配置结构一致**：所有单位的属性配置使用相同的字段结构
3. **可扩展性**：新属性的添加只需扩展配置结构，无需修改每个单位的配置代码
4. **配置驱动**：属性值通过配置文件（而非硬编码）驱动，便于调整和平衡

### 配置数据结构

所有单位属性使用统一的配置结构：

```yaml
UnitAttributes:
  # 基础属性
  maxHealth: 数值        # 生命值（必须是10的倍数）
  attackPower: 数值       # 攻击力
  attackCooldown: 数值    # 攻击冷却（秒）
  attackRange: 数值       # 攻击范围（像素）
  pursuitRange: 数值      # 追击范围（像素）
  armorValue: 数值        # 护甲值
  bodySize: 数值          # 体型大小
  
  # 自动计算的衍生属性
  isRanged: true/false    # 是否为远程单位（影响追击范围）
  
  # 扩展属性预留
  # 未来可添加更多属性，如：移动速度、特殊技能等
```

### 配置系统优势

1. **统一管理**：所有单位配置在统一的数据结构中，便于查找和维护
2. **易于扩展**：添加新属性只需扩展配置结构，无需修改大量代码
3. **平衡调整**：调整属性数值只需修改配置文件，无需重新编译
4. **数据验证**：统一的配置接口便于进行数据验证（如生命值必须是10的倍数）
5. **配置复用**：可以定义基础配置，通过继承或合并创建变体

### 配置加载流程

1. 加载配置文件（如YAML或JSON格式）
2. 验证配置数据（数据类型、取值范围等）
3. 应用配置到单位实例
4. 根据单位类型自动计算衍生属性（如远程单位设置追击范围=攻击范围）

### 实现说明

**注意**：本文档为逻辑设计文档，具体实现细节将在代码中完成。配置系统设计确保：
- 所有22种单位使用相同的配置结构
- 新增单位时只需添加配置数据
- 新增属性时只需扩展配置结构
- 属性数值易于调整和平衡

## 属性说明

### 核心属性
- **生命值**：单位最大生命值，必须是10的倍数
- **攻击力**：单位每次攻击造成的伤害
- **攻击冷却**：两次攻击之间的间隔时间（秒）
- **攻击范围**：单位可以攻击的最大距离（像素）
- **追击范围**：敌人进入此范围后开始追击
  - 远程单位：追击范围 = 攻击范围
  - 近战单位：追击范围 = 攻击范围 × 3
- **护甲值**：减少受到的伤害，公式：实际伤害 = max(1, 攻击力 - 护甲值)
- **体型大小**：用于计算击退距离和碰撞检测

### 伤害计算
- **实际伤害** = max(1, 攻击力 - 护甲值)
- **保底伤害**：即使护甲很高，也会受到至少1点伤害

### 击退系统
- 击退距离取决于攻击者和被攻击者的体型大小差值
- 体型大的单位不易被击退，体型小的单位容易被击退

## 属性平衡分析

### 性价比平衡原则

1. **训练成本 vs 战斗能力**：
   - 初始单位（哥布林）：战斗力最弱，但无成本
   - 进化单位（兽人）：成本较高，获得基础战斗能力
   - 训练单位：根据训练时间调整属性
   - 购买装备单位：投资装备，获得显著提升
   - 晋级单位：特殊机制，属性飞跃式提升

2. **DPS平衡**：
   - 功能类单位：DPS 2.5-9.23（弱，主要用于工作）
   - 战斗类单位：DPS 6.67-38.89（强，提供多样性）
   - 角斗士系列：DPS最高，但护甲极低，平衡输出和生存

3. **生存能力平衡**：
   - 护甲值设计遵循"高攻击力低护甲"原则
   - 角斗士：高攻击力，低护甲，定位输出
   - 重装战士：中等攻击力，超高护甲，定位坦克
   - 远程单位：中等生命值，靠射程优势生存

4. **特殊单位设计**：
   - 召唤单位：随机性补偿，整体属性高于一般单位
   - 萨满：攻击力低但功能性强，提供治疗和仪式

### 阵容搭配建议

**经济型阵容**（低成本高性价比）：
- 哥布林（挖矿）
- 哥布林战士（防卫）
- 兽人（主力）

**平衡型阵容**：
- 地精（建造）
- 兽人战士（坦克）
- 兽人猎手（远程）
- 兽人萨满（治疗）

**高级阵容**（高投入高回报）：
- 兽人重装战士（坦克）
- 兽人劲弩猎手（远程）
- 兽人冠军（输出）
- 野兽人召唤（全面）

## 总结

所有22种单位的属性已经平衡设计，遵循以下原则：
1. **生命值均为10的倍数**
2. **护甲值遵循"高攻击力低护甲"原则**（25攻击对应800生命和3护甲）
3. **性价比平衡**（包括时间和资源成本）
4. **功能差异化明显**（远程/近战/坦克/输出/辅助）
5. **击退系统为体型大小提供战术意义**
6. **追击范围设计合理**（远程=攻击范围，近战=攻击范围×3）
